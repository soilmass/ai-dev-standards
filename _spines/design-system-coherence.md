# Design-System Coherence & Front-End Quality Spine

Front-end quality is not a layer — it's whether the *whole site* looks, reads, and behaves like one coherent product across every page, every breakpoint, and every asset, and it is decided in design, built in markup, and proven by a site-wide gate. A landing page that passes in isolation tells you nothing about the 404 in the footer, the drifted header on page three, or the route that ships a second `<h1>` and no description. This spine doesn't duplicate the layer rules; it shows **where front-end quality bites in each layer**, owns the cross-cutting obligations no single page-level check can see (whole-site integrity, design-system grounding, the borrow-discipline), and ties the design contract (tokens + HIG grounding) to the site-integrity gate that verifies it everywhere.

## Cross-cutting rules owned here

1. **The unit of front-end quality is the whole site, not the page.** Every internal, nav, and footer link resolves (0 404s); header, footer, and primary nav are byte-for-byte the same component across every route; the sitemap and the actual routes are mutually exhaustive (no orphan page, no dead sitemap entry); and every breadcrumb names a real parent. These are properties no single-page check can see — they are owned by `03-design/multi-page-site-coherence.md` and proven site-wide by the site-integrity gate (`stacks/nextjs-default/project-config/scripts/check-site-integrity.example.sh`, run in CI). A green landing page over three site-wide 404s is a failing site.
2. **The design system is grounded in a named human-interface guideline, and borrowing from it is disciplined.** Tokens, components, and interaction patterns trace to a specific authority — **Apple HIG**, **Material Design 3**, or the **ARIA Authoring Practices Guide** — chosen per surface and named in `03-design/design-tokens-and-hig.md`. You adopt a platform's *pattern* (the dialog's focus-and-dismiss contract, the navigation hierarchy) without cargo-culting its *chrome* onto a different platform; a finding that a widget violates its grounding cites the **specific rule** (e.g. "APG dialog: focus must return to the invoking control on close"), not "feels off." Bad borrowing — iOS chrome on a web app, a half-implemented MD3 component — is worse than no borrowing.
3. **Tokens are the single source of visual truth; raw values are violations.** Color, spacing, type scale, radii, and motion are a DTCG-shaped token taxonomy defined once and referenced everywhere (`03-design/design-tokens-and-hig.md`, `03-design/ui-design-system.md`). A bare hex or a magic pixel in component code is a defect the no-bare-hex / token-completeness lint catches; theming and dark mode work through token aliasing, never per-component conditionals. One styling system, applied one way — a second approach accreting per feature is the drift this spine exists to stop.
4. **Every route meets the same baseline, not just the first five.** Each page in the sitemap independently clears the a11y floor, carries a **unique** `<title>` and meta description, has exactly **one** `<h1>`, and ships complete metadata (canonical, OG, viewport) — verified on *every* route, never a sampled subset (`03-design/per-page-quality-baseline.md`, gated by the per-route extension of `05-verification/a11y-perf-gates.md`). Sampling the homepage and declaring the site good is how the second, third, and fourth pages rot unseen.
5. **Layout is responsive on a fixed, mobile-first scale — and proven at every breakpoint.** Breakpoints are declared and ordered mobile-first; spacing and grid follow the token scale, not ad-hoc values; and the layout is visually regression-tested at the standard widths (320 / 768 / 1024 / 1280) so a fix that reflows the desktop grid can't silently break the phone view (`03-design/responsive-layout-discipline.md`, `05-verification/visual-regression.md`). "Looks fine on my monitor" is not a viewport.
6. **Graphics and assets cohere and resolve.** OG/social images, icons, and illustrations share one visual language (the same grid, stroke weight, corner language, and palette as the tokens); every referenced image resolves (no 404'd OG card); responsive images declare correct `sizes` and an efficient format; and `alt` text is meaningful for content images and empty for decorative ones (`03-design/graphics-asset-cohesion.md`). A presence-only check ("an OG tag exists") passes while the image it points at 404s — cohesion and resolution are both required.

## Standards basis

- **Apple Human Interface Guidelines** (developer.apple.com/design/human-interface-guidelines/) and **Material Design 3** (m3.material.io) — the platform interface authorities behind rule 2's grounding-and-borrow-discipline; a design-system decision names which one (and which surface) it follows.
- **WAI-ARIA Authoring Practices Guide (APG)** (w3.org/WAI/ARIA/apg/) — the per-widget interaction grounding that rule 2 requires findings to cite by name; it is the same APG the accessibility spine (`_spines/accessibility.md`) owns for the keyboard/role/state contract, viewed here through the design-system-coherence lens.
- **W3C Design Tokens Format Module** (DTCG; `.tokens` / `application/design-tokens+json`) — the vendor-neutral token taxonomy (groups, aliases, types) behind rule 3: tokens defined once, referenced everywhere, theming via aliasing.
- **WCAG 2.2 Level AA** (w3.org/TR/WCAG22/) — the per-page a11y floor in rule 4 and the contrast/target-size/focus criteria that token and layout decisions must satisfy; conformance is owned by `05-verification/a11y-perf-gates.md` and the accessibility spine, and this spine requires it on *every* route, not the sampled few.
- **Core Web Vitals — CLS** (web.dev/articles/vitals) and the **responsive-images** spec (`srcset`/`sizes`, modern formats) — the layout-stability and image-correctness basis for rules 5–6; the lab gate is owned by `05-verification/a11y-perf-gates.md` (the performance spine, `_spines/performance.md`, owns the budget framing).
- Builds on the new front-end-quality standards it ties together — `03-design/multi-page-site-coherence.md`, `03-design/per-page-quality-baseline.md` *(gated by the per-route extension of `05-verification/a11y-perf-gates.md`)*, `03-design/design-tokens-and-hig.md`, `03-design/responsive-layout-discipline.md`, `03-design/graphics-asset-cohesion.md` (owned in design), and the site-integrity gate (`stacks/nextjs-default/project-config/scripts/check-site-integrity.example.sh`) that proves them site-wide. It overlaps the accessibility spine (`_spines/accessibility.md`) on the WCAG/APG floor and the performance spine (`_spines/performance.md`) on CLS/images by design — those spines own the a11y and budget halves; this one owns the design-system-coherence and whole-site-integrity connective tissue across them.

## Enforcement
- Mechanism: CI job
- Config: stacks/nextjs-default/project-config/scripts/check-site-integrity.example.sh (site-wide link/chrome/sitemap integrity, allowed-404=0) + stacks/nextjs-default/ci/lighthouserc.json + stacks/nextjs-default/ci/pr.yml (per-page a11y/perf/SEO baseline on every sitemap route) + stacks/nextjs-default/lint-config/biome.json (no-bare-hex / token-completeness + a11y markup) + stacks/nextjs-default/project-config/playwright.config.example.ts (multi-viewport visual regression) — the mechanical subset; each layer row above names its own gate.
- Fallback if unenforceable: n/a — link integrity, per-page baseline, token/markup lint, and multi-viewport regression are CI-gated; the judgment pieces (HIG borrow-discipline, graphics/illustration cohesion, alt-text quality) ride the design-system and graphics-cohesion review items aggregated in `05-verification/code-review-standard.md` §E (owned by `03-design/design-tokens-and-hig.md` and `03-design/graphics-asset-cohesion.md`).

## Bootstrap
- What new-project.sh injects for this standard: nothing additional — the site-integrity check, the per-route Lighthouse/axe baseline, the no-bare-hex / token-completeness lint, and the multi-viewport Playwright config that the design and verification standards inject are this spine's enforcement surface.
